# Step 09. 이벤트 확인 페이지 UI 개편

## 목적

`/events` 페이지에서 메인 회의실 입퇴장과 소회의실 관련 이벤트를 더 빨리 구분할 수 있게 화면 구조를 바꾼다.

이번 단계는 시각 개선과 설명 보강이 목적이다.

## 변경 내용

- 전체 색상을 푸른 계열로 변경
- 상단에 요약 카드 추가
  - 현재 표시 이벤트 수
  - 메인 회의실 또는 일반 입퇴장 수
  - 소회의실 확정 이벤트 수
  - 소회의실 이동 추정 수
- 각 이벤트를 카드형으로 재구성
- 카드 상단에 이벤트 종류와 회의실 판정 배지 추가
- 메인/소회의실 판정을 한글 설명 문장으로 표기
- raw payload를 열기 전에도 현재 판정 근거를 짧게 읽을 수 있게 구성

## 판정 표시 방식

- `main_join`
  - 기본 메인 회의실 입장
- `meeting_left`
  - 회의 완전 퇴장
- `temporary_breakout_exit`
  - 소회의실 입장을 위한 임시 퇴장
- `breakout_join_inferred`
  - 이전 임시 퇴장 기록을 근거로 추론한 소회의실 입장

## 결과

이제 `/events` 페이지에서 raw payload를 열지 않아도 아래를 바로 읽을 수 있다.

- 지금 이벤트가 메인 입장인지
- 회의 완전 퇴장인지
- 소회의실 입장을 위한 임시 퇴장인지
- 이전 기록을 근거로 소회의실 입장으로 볼 수 있는지
