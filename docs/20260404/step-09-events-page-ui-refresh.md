# Step 09. 이벤트 확인 페이지 UI 개편

## 목적

`/events` 페이지에서 메인 회의실 입퇴장과 소회의실 관련 이벤트를 더 빨리 구분할 수 있게 화면 구조를 바꾼다.

이번 단계는 시각 개선과 설명 보강이 목적이다.

## 변경 내용

- 전체 색상을 푸른 계열로 변경
- 상단에 요약 카드 추가
  - 현재 표시 이벤트 수
  - 메인 회의실 또는 일반 입퇴장 수
  - 소회의실 확정 이벤트 수
  - 소회의실 이동 추정 수
- 각 이벤트를 카드형으로 재구성
- 카드 상단에 이벤트 종류와 회의실 판정 배지 추가
- 메인/소회의실 판정을 한글 설명 문장으로 표기
- raw payload를 열기 전에도 현재 판정 근거를 짧게 읽을 수 있게 구성

## 판정 표시 방식

- `main_or_unknown`
  - 화면에서는 `메인 회의실 또는 일반 입퇴장`으로 표시
  - 설명 문구도 `메인 회의실 입장`, `메인 회의실 퇴장`처럼 한글로 노출
- `breakout`
  - 화면에서는 `소회의실`로 표시
  - payload에 직접 들어온 소회의실 정보를 함께 설명
- `breakout_left`
  - 화면에서는 `소회의실 퇴장`으로 표시
  - `leave_reason`에 `left the meeting to join breakout room` 문구가 있으면 이 규칙을 우선 적용
- `breakout_transition`
  - 화면에서는 `소회의실 이동 추정`으로 표시
  - `leave_reason` 기반 추정이라는 점을 같이 설명

## 결과

이제 `/events` 페이지에서 raw payload를 열지 않아도 아래를 바로 읽을 수 있다.

- 지금 이벤트가 입장인지 퇴장인지
- 메인 회의실로 볼지 소회의실로 볼지
- 소회의실 확정인지, 이동 추정인지
